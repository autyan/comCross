## Checklist

### 1. Current Release Development State

v0.6.0 has completed user acceptance for the current message-storage and
message-viewer scope:

- optional per-session Archive and archive history mode;
- live/archive viewer data-source separation;
- Plain, Slim, and Detailed display modes;
- cancellable viewer search and navigation;
- large-message display improvements through AvaloniaEdit;
- per-session display mode persistence;
- session tree and workload tab UI polish;
- send-tool eligibility and per-session draft handling;
- host shutdown cancellation cleanup;
- first static website source.

### 2. Mandatory Technical Debt

No release-blocking technical debt is currently known after local verification.

Accepted deferrals:

- persistent full-text index;
- text-level search highlight across all display modes;
- hex search;
- decoded-field search and persistence;
- detailed single-row copy commands;
- lightweight analysis dashboard;
- archive import, replay, backfill, and portable archive export.

These items are assigned to later scopes, with the near-term hardening scope
documented in:

```text
docs/roadmap/v0.6.1-scope.md
```

### 3. Documentation Accuracy

The prerelease branch now contains release-state documentation for v0.6.0:

- `docs/release/notes/v0.6.0.md`
- `docs/release/changelog/v0.6.0.md`
- `docs/roadmap/v0.6.0-scope.md`
- `docs/roadmap/v0.6.1-scope.md`
- `README.md`
- `website/`

Localized README files are intentionally deferred to the post-release
documentation pass. The English README and release documents remain
authoritative.

### 4. GitHub Actions Risk

Known release workflow risks:

- Full packaging verification has not yet been run from this review branch.
- Formal releases still require signing secrets when `require_signing=true`.
- The Windows code-signing certificate is currently self-signed and documented
  as a limitation.
- The release workflow requires `release_notes_path` for final non-draft
  releases.
