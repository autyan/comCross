# ComCross v0.5.0 Release Notes

ComCross v0.5.0 focuses on release engineering, signed installable packages,
and a clearer supported operating system baseline.

## Package Assets

The official GitHub Release should include:

- `ComCross-0.5.0-win-x64.msi`
- `ComCross-0.5.0-win-arm64.msi`
- `comcross_0.5.0_amd64.deb`
- `comcross_0.5.0_arm64.deb`
- `comcross-0.5.0-1.x86_64.rpm`
- `comcross-0.5.0-1.aarch64.rpm`
- `ComCross-0.5.0-linux-x64.AppImage`
- `ComCross-0.5.0-linux-arm64.AppImage`
- `SHA256SUMS`
- `SHA256SUMS.asc`

Portable `.tar.gz` and `.zip` archives are not default release assets. Users
who need bare publish outputs should build them locally from source.

## Supported Operating Systems

The v0.5.0 package support baseline is:

| OS | Minimum version | Architecture | Package |
|---|---|---|---|
| Windows | Windows 10 22H2 / Windows 11 22H2 | x64, ARM64 | MSI |
| Ubuntu | 22.04 LTS | x64, ARM64 | DEB, AppImage |
| Debian | 12 | x64, ARM64 | DEB, AppImage |
| Fedora | 40 | x64 | RPM, AppImage |

This is a baseline compatibility target. ComCross does not yet have enough
testing resources to fully validate every supported OS version, desktop
environment, graphics stack, and hardware configuration.

Other Linux distributions may work, but they are outside the formal support
commitment. AppImage packages are the recommended fallback for these users.

ComCross does not currently provide official macOS packages.

## Verification

Verify downloaded assets with:

```bash
sha256sum -c SHA256SUMS
gpg --verify SHA256SUMS.asc SHA256SUMS
```

Windows MSI packages are signed during the release workflow when signing
material is configured. Linux packages and AppImages are covered by
`SHA256SUMS`; the checksum file is signed for formal releases.

## Upgrade Notes

The v0.4.0+ workspace format is not compatible with old v0.3 session state.
Existing sessions from older development builds should be recreated.

## Release Highlights

- GitHub Actions release workflow for signed package generation.
- Windows MSI packages for x64 and ARM64.
- Linux DEB/RPM packages for mainstream distributions.
- Linux AppImage packages as fallback packages.
- Unified `SHA256SUMS` generation across final release assets.
- Signed official plugin packages in release outputs.
- Release build output is clean of compiler and analyzer warnings at the time
  of the v0.5.0 release workflow validation.

## Known Limitations

- The Windows signing certificate used during current validation is a
  self-signed test code-signing certificate, not a publicly trusted commercial
  certificate.
- Non-mainstream Linux distributions are not formally supported.
- AppImage packages are provided as fallback packages, not as a guarantee that
  every Linux desktop environment is fully validated.
- macOS packages are not provided in this release.
