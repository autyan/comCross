# v0.4.0 Changelog

## Release Goal

Make ComCross stable enough for wider use by converging the architecture,
plugin boundary, session model, message model, and core communication workflows.

## Architecture

- Established the main program and bus plugins as a producer/consumer boundary.
- Moved plugin-specific facts behind contracts instead of having Core or Shell
  infer plugin-private semantics.
- Cleaned up plugin runtime coordination, capability handling, and IPC-facing
  responsibilities.
- Added AI collaboration rules and repository guardrails to preserve the new
  architecture boundary.

## Session And Message Model

- Stabilized session metadata consumption in Core and Shell.
- Added bounded message attributes for plugin-supplied message facts.
- Added message attribute display and search support.
- Made session deletion explicit: deleting a session also deletes the
  corresponding session data.
- Removed legacy migration paths for old v0.3 development workspace/session
  data.

## Communication Workflows

- TCP session hierarchy is provided by plugin relationship metadata.
- UDP no longer creates one child session per remote endpoint.
- UDP target selection supports single-entry, multi-target communication
  workflows.
- Serial port scanning is owned by the serial plugin and consumed by the main
  program through plugin capabilities/actions.
- Serial refresh is routed through the plugin action path.

## Shell UX

- Reused the same connection parameter UI/VM path for quick connect and session
  details.
- Improved reconnect behavior and connection parameter backfill.
- Added a lightweight quick command picker with editable defaults, pinned
  commands, search/select/send flow, and improved hit targets.
- Fixed popup focus/closing behavior and composite input visual framing.
- Improved session detail, delete dialog, and i18n coverage.

## Release Engineering

- Published portable archive packages for Windows and Linux on x64 and ARM64.
- Documented the remaining package distribution gaps for the next release.

## Compatibility

- v0.4.0 becomes the stable baseline after the v0.3 development line.
- Old v0.3 development workspace/session migration is not retained.
- Back up old data before upgrading from pre-v0.4 builds.

## Known Limitations

- Windows MSI packages are not provided in this release.
- DEB, RPM, and AppImage packages are not provided in this release.
- Package signing and automated checksum/signature generation are not provided
  in this release.
- GitHub Actions release automation is introduced in v0.5.0.
