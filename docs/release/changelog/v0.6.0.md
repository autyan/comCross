# v0.6.0 Changelog

## Release Goal

Establish the message storage and viewer foundation for large communication
sessions while keeping transport plugins as producers of raw communication
facts.

## User-Facing Changes

- Added session-level Archive as an optional long-term history store.
- Added Archive history mode alongside live message viewing.
- Added explicit live/archive data-source handling in the message viewer.
- Added Plain, Slim, and Detailed display modes.
- Reworked Plain and Slim rendering around aggregate text selection.
- Added Slim RX/TX gutter rendering while keeping payload text selectable.
- Added cancellable message search as a viewer navigation workflow.
- Added search over current available message data with direction and
  attribute-aware scope.
- Added clearer return-to-latest behavior after search or manual history
  browsing.
- Added per-session display mode persistence.
- Added separate UI and message font settings, including message font size.
- Disabled message sending for session types that cannot transmit directly.
- Preserved per-session draft message input.
- Improved session tree hierarchy layout and listener/child session alignment.
- Improved warning styling for disconnect actions.
- Improved workload tab sizing for English and localized default labels.
- Published the first official static website under `website/`.

## Message Viewer And Storage

- Plain mode renders raw message text as a continuous stream.
- Slim mode keeps frame boundaries through an RX/TX gutter while rendering
  payload text using the same string rules as Plain mode.
- Detailed mode keeps one row per frame and escapes invisible string payload
  characters so frame structure remains stable.
- STR payload rendering treats CR/LF as line structure in Plain and Slim modes.
- HEX payload rendering wraps as text and does not attempt to interpret CR/LF.
- Archive history mode and live spool mode share the same viewer-facing frame
  model.

## Release Engineering

- Upgraded the Shell UI stack to the new Avalonia baseline used by the
  prerelease branch.
- Adopted AvaloniaEdit for aggregate message text rendering.
- Fixed host IPC shutdown cancellation so Ctrl+C exits do not print expected
  `OperationCanceledException` stacks.
- Added AI collaboration rules for message viewer rendering and CR/LF display
  semantics.
- Continued using the v0.5 release packaging workflow for Windows MSI, Linux
  DEB/RPM, Linux AppImage, checksums, and optional signing.

## Compatibility

- The v0.4.0+ workspace format remains incompatible with old v0.3 development
  state. Existing sessions from older development builds should be recreated.
- Session Archive is not a portable archive package format.
- Live spool and archive internal storage formats are implementation details.
- macOS packages are not provided.
- Non-mainstream Linux distributions remain outside the formal support
  commitment.

## Documentation

- Added v0.6.0 release notes and changelog.
- Added v0.6.0 prerelease review record.
- Added v0.6.1 scope as the next development stage.
- Updated README and website status to describe v0.6.0 as the current release
  line.
- Kept v0.6.0 roadmap documents as authoritative scope and acceptance
  references.

## Known Limitations

- Persistent full-text indexing is not included.
- Text-level search highlight in all viewer modes is deferred to a later
  release.
- Hex search is not included.
- Decoded-field search and decoded-field persistence are not included.
- Lightweight analysis dashboards are deferred.
- Detailed single-row copy commands are deferred.
- Archive import, archive replay, archive backfill, and portable archive export
  are not included.
- Localized README files may lag the English release-state update until the
  post-release documentation pass.
