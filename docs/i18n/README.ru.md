ComCross — это кроссплатформенный набор инструментов для встроенных
коммуникационных сценариев через serial, TCP и UDP.

## Статус

ComCross пока находится до стабильного периода совместимости. Несовместимые
изменения допустимы, если они исправляют архитектурные границы, runtime
директории, структуру плагинов или правила упаковки. Такие изменения должны
быть задокументированы.

## Основные возможности

- Serial, TCP и UDP адаптеры поставляются как изолированные плагины.
- Управление сессиями и workloads с сохраняемыми дескрипторами.
- Поисковый RX/TX поток сообщений с ограниченными атрибутами кадров.
- Страницы настроек плагинов и UI-состояние, создаваемое плагинами.
- Встроенная локализация Shell на английском и упрощенном китайском.

## Быстрый старт для разработки

```bash
dotnet restore
dotnet build ComCross.sln --no-restore
dotnet run --project src/Shell/ComCross.Shell.csproj
dotnet test ComCross.sln --no-build
```

## Важные замечания

- Будущая пользовательская точка входа планируется как `ComCross.Startup`.
- Публичные Windows-релизы сначала могут использовать самоподписанный тестовый
  сертификат. Windows не доверяет такому сертификату по умолчанию.
- Перед обходом предупреждений ОС проверяйте checksums и подпись релиза.
- Подпись официальных пакетов плагинов будет отделена от Windows code signing
  и подписи release artifacts.
