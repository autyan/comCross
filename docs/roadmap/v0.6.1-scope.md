# ComCross v0.6.1 Scope

## Theme

v0.6.1 is a hardening release for the v0.6.0 message workflow. It should make
search, message viewing, export, and archive/spool behavior feel complete and
predictable before decoder, protocol-analysis, or dashboard work resumes.

## Scope 1: Search Hardening

Required capabilities:

- Text-level search highlight in Detailed mode.
- Text-level search highlight in Plain and Slim aggregate text modes.
- Consistent search navigation behavior across Plain, Slim, and Detailed modes.
- Documented lightweight search syntax for payload, direction, timestamp, and
  attributes.
- Clear search states for no match, canceled, source unavailable, archive
  disabled, and data evicted.
- Regression scenarios for search over large live and archive data sets.

Out of scope:

- Persistent full-text index.
- Decoded-field search.
- Protocol parse status search.
- Cross-workspace or cross-session global search.

## Scope 2: Message View Interaction

Required capabilities:

- Detailed single-frame copy command.
- Copy payload and copy raw data actions where the selected frame is available.
- Stable Plain/Slim copy behavior that follows the rendered text closely.
- Documented STR/HEX/invisible-character rendering rules for all display modes.
- Regression scenarios for viewport sizing, auto-scroll, return-to-latest, and
  display-mode switching.

Out of scope:

- Rich row action framework.
- Editable message history.
- Protocol-decoded field copy.

## Scope 3: Export Reliability

Required capabilities:

- Plain, Slim, and Detailed export rules aligned with viewer rendering.
- Export from live spool and archive sources with equivalent behavior where the
  source is available.
- Export progress, cancellation, and failure reporting.
- Large export execution without loading all frames into Shell memory.
- Optional export metadata header with session and source summary.

Out of scope:

- Portable archive package export.
- Archive import.
- Replay package export.
- Partial range export UI unless needed for implementation safety.

## Scope 4: Archive And Spool Reliability

Required capabilities:

- Clear user-visible states for archive disabled, archive error, data evicted,
  and source unavailable.
- Session deletion behavior fully verified for spool and archive data.
- Archive corruption, schema mismatch, and missing-file diagnostics.
- Storage calibration and conservative fallback behavior reviewed against real
  user workflows.
- Settings terminology cleanup for Application Logs, Session Logs, and Session
  Archive.

Out of scope:

- Archive compaction UI.
- Archive migration/import.
- Orphan archive retention.

## Scope 5: Release And Documentation Automation

Required capabilities:

- Prerelease review template or checklist reuse for future releases.
- Release notes and changelog presence checks for final release branches.
- README current-version update checklist.
- Website latest manifest update checklist.
- Known-limitations section kept aligned with release notes and roadmap.

Out of scope:

- Full documentation site migration.
- Full localized documentation refresh automation.

## Explicitly Deferred Beyond v0.6.1

- Decoder and CPDL implementation.
- Lightweight analysis dashboard.
- Decoded-field persistence.
- Protocol inference.
- Multi-session correlation.
- Persistent full-text index.
- Portable archive import/export package.

## Release Acceptance

v0.6.1 should be considered complete when:

- Search highlight and navigation are reliable in all message display modes.
- Message copy/export behavior is documented and matches viewer rendering.
- Archive/spool error states are visible and actionable.
- Large data workflows remain windowed or streaming and do not regress into
  Shell-wide full-history memory requirements.
- Release documentation checks are explicit enough that v0.6.1 release docs
  cannot be missed during prerelease review.
