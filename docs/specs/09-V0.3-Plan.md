# V0.3 Plan

## Goals
- Build an application logging system distinct from workspace logs.
- Validate plugin mechanism with representative modules.
- Provide platform packaging for GitHub releases.
- Complete the command system, export tooling, and configuration bindings.

## Scope

### 1) Application Logging (NLog)
- Log outputs: file only
- Default format: txt
- Optional format: json
- Separation: application logs vs workspace/business logs
- Exception model: standardized categories and severity mapping

### 2) Command System
- Universal command model: id/name/payload/type/encoding/CRLF
- Preset + per-session + global scopes
- One-click send and hotkey support
- User-defined commands
  - UI: list/create/edit/group/sort
  - Import/export (JSON)

### 3) Plugin System Full Loop
- Plugin loading/validation/lifecycle
- Plugin notification subscription (language change as notification)
- Official plugins must implement i18n handling
- Plugin exception isolation
- Plugin host process per plugin
- Plugin manager UI (enable/disable/permissions)
- Validation plugins
  - Statistics plugin (general purpose)
  - Hardware protocol plugin (common device/protocol support)
  - Modular flow plugin (placeholder + basic functionality for future DSL/JS)

### 4) Export Tooling
- Formats, ranges, filtered export, default paths

### 5) Configuration Completion
- Bind configuration to behavior (timestamp format, auto-scroll, etc.)

### 6) Engineering & Stability
- Log performance
- Exception handling
- Notification reliability

### 7) Packaging & Release
- Windows: MSI installer
- Linux: DEB and RPM
- Release artifacts for GitHub with consistent naming
- Runtime baseline selection (target .NET 8 LTS for releases)

## Milestones
- M1 Application logging system (NLog integration + exception model)
- M2 Command system + user-defined commands
- M3 Plugin system validation set (3 plugins) + manager UI
- M4 Export + configuration bindings
- M5 Packaging pipeline (MSI/DEB/RPM)
